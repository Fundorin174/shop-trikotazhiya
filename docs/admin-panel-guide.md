## 1. Вход в систему

1. Откройте страницу `/admin`
2. Введите **Email** и **Пароль** администратора  
   _(по умолчанию: `admin@trikotazhiya.ru`, пароль задан при установке — см. `.env`)_
3. Нажмите **«Войти»**

При успешной авторизации откроется панель управления.

> **Ошибки входа:**
> - «Неверный email или пароль» — проверьте учётные данные
> - «Токен не получен» — сервер Medusa работает, но вернул пустой ответ; перезапустите бэкенд

Для выхода нажмите кнопку **«Выйти»** в правом верхнем углу.

## 2. Обзор интерфейса

После входа доступны три вкладки:

| Вкладка | Назначение |
|---------|------------|
| **Товары** | Просмотр, сортировка, быстрое редактирование цены/запаса, удаление, переход в полное редактирование |
| **Импорт** | Массовая загрузка товаров из JSON-файла |
| **+ Добавить** | Ручное создание нового товара через форму (также используется для полного редактирования) |

## 3. Таб «Товары» — список товаров

Таблица отображает все товары магазина с ключевыми параметрами.

### 3.1. Сортировка

Клик по заголовку любого столбца переключает сортировку:
- Первый клик → сортировка **по возрастанию** (↑)
- Второй клик → сортировка **по убыванию** (↓)
- Значок ↕ означает, что столбец не активен

Доступные столбцы для сортировки:

| Столбец | Что сортирует |
|---------|---------------|
| **SKU** | Артикул (числовая сортировка) |
| **Название** | Алфавитная сортировка (А–Я) |
| **Тип** | Тип ткани по алфавиту |
| **Цена** | Числовая сортировка (с учётом единицы измерения) |
| **Статус** | Опубликован / Черновик |
| **Запас** | По количеству на складе |
| **Дата** | По дате создания |

По умолчанию товары отсортированы по **SKU** по возрастанию.

Кнопка **«↻ Обновить»** позволяет перезагрузить список с сервера.

### 3.2. Быстрое редактирование цены

1. **Дважды кликните** по ячейке с ценой
2. Введите новую цену в **рублях**:
   - Для метровых тканей — цена за **1 погонный метр**
   - Для штучных товаров — цена за **1 штуку**
3. Нажмите **Enter** или кнопку **✓**
4. Появится подтверждение: **«Сохранить?»** → нажмите **«Да»**
5. Для отмены нажмите **Escape**, кнопку **✕** или **«Нет»**

### 3.3. Быстрое редактирование запаса

1. **Дважды кликните** по ячейке с запасом
2. Введите новое значение:
   - Для метровых тканей — количество в **метрах** (напр. `300` = 300 м)
   - Для штучных товаров — количество в **штуках**
3. Нажмите **Enter** или **✓** → подтвердите: **«Да»**
4. Для отмены: **Escape**, **✕** или **«Нет»**

> **Примечание:** Значение запаса `0` выделяется **красным цветом**.  
> Прочерк (—) означает, что запас не настроен.

### 3.4. Удаление товара

1. Нажмите иконку **🗑 (корзина)** в столбце «Действие»
2. Появится подтверждение: **«Удалить?»** с кнопками **«Да»** / **«Нет»**
3. Нажмите **«Да»** для удаления

> ⚠️ **Внимание:** Удаление необратимо. Товар удаляется из базы данных Medusa вместе со всеми вариантами.

### 3.5. Редактирование товара

1. Нажмите иконку **✏️ (карандаш)** в столбце «Действие»
2. Будет загружена полная информация о товаре (спиннер «Загрузка данных товара...»)
3. Автоматически откроется вкладка **«Редактировать товар»** с заполненной формой
4. Внесите изменения в нужные поля
5. Нажмите **«Сохранить изменения»**

**Особенности режима редактирования:**
- Поле **SKU** заблокировано — его нельзя изменить (артикул является уникальным идентификатором)
- Существующие фотографии отображаются с подписью «Текущие фото» — их можно удалить, наведя курсор и нажав ✕
- При удалении фотографий из товара файлы автоматически удаляются с сервера
- Можно добавить новые фотографии — они будут добавлены к оставшимся существующим
- Кнопка **«Отмена»** возвращает на вкладку «Товары» без сохранения
- При сохранении старый товар удаляется и создаётся заново с обновлёнными данными

## 4. Таб «+ Добавить» — создание товара

Форма создания содержит несколько секций. Поля, отмеченные звёздочкой (*), обязательны.

### 4.1. Основные поля

| Поле | Обязательное | Описание | Пример |
|------|:-----------:|----------|--------|
| **Название** | ✅ | Полное название товара | `Кулирка пенье серый меланж` |
| **SKU** | ✅ | Уникальный артикул | `00301` |
| **Тип ткани** | ✅ | Выбор из списка (12 типов) | `Кулирка` |
| **Состав** | ✅ | Текстовое описание состава | `100% хлопок` |
| **Ширина (см)** | ✅ | Ширина полотна в сантиметрах | `180` |
| **Единица измерения** | — | «Метры (погонный метр)» или «Штуки» | По умолч.: метры |

### 4.2. Цена и запас

| Поле | Обязательное | Описание | Пример |
|------|:-----------:|----------|--------|
| **Цена (₽)** | ✅ | Цена в рублях за 1 метр или за штуку | `1100` |
| **Запас** | — | Для метровых: в **сантиметрах** (30000 = 300 м). Для штучных: в штуках | `30000` |

> **Важно:** Запас метровых тканей указывается в **сантиметрах**, не в метрах!  
> Например: 300 метров = `30000`, 50 метров = `5000`.

### 4.3. Скидка

Оба поля необязательны. Можно указать одно из них или оба:

| Поле | Описание | Пример |
|------|----------|--------|
| **Скидка (%)** | Процент скидки (0–100) | `10` |
| **Скидка (₽)** | Фиксированная скидка в рублях | `100` |

Если скидки нет — оставьте поля пустыми.

### 4.4. Изображения

- Нажмите **«Добавить фото»** для выбора файлов
- Можно загрузить **несколько** изображений за раз
- **Первое** изображение автоматически становится **обложкой** (отмечено синим бейджем «Обложка»)
- Рекомендуемое разрешение: **800×600 px**
- Максимальный размер файла: **300 КБ**

> **Если фото слишком большое:**  
> Появится подсказка со ссылкой на сервис сжатия: [iLoveIMG — Сжать изображение](https://www.iloveimg.com/ru/compress-image)

Для удаления загруженного фото наведите курсор и нажмите красную кнопку **✕** в углу.

### 4.5. Дополнительные поля

Раскрываются по клику на секцию **«Дополнительно»**. Все поля необязательны:

| Поле | Описание | Пример |
|------|----------|--------|
| **Handle (URL-слаг)** | URL товара. Если пустой — генерируется из названия | `kulirka-penye-seryj-melanzh` |
| **Подзаголовок** | Доп. текст под названием | `Пенье, 180 г/м²` |
| **Цвет** | Название цвета | `Серый меланж` |
| **Цвет (HEX)** | HEX-код цвета | `#808080` |
| **Качество** | Качество ткани | `Пенье` |
| **Страна** | Страна производства | `Турция` |
| **Коллекция** | Назначение коллекции | `Базовая` |
| **URL видео** | Ссылка на видео-обзор | `https://youtube.com/...` |
| **Мин. заказ** | Минимальный объём заказа (м или шт) | `0.5` |
| **Шаг заказа** | Шаг увеличения в корзине (м или шт) | `0.1` |
| **Краткое описание** | Для карточки товара | `Мягкая кулирка для детской одежды` |
| **Описание** | Подробное описание | Текст с описанием характеристик |
| **Полное описание (HTML)** | Расширенное описание с HTML-разметкой | `<p>Ткань <b>высокого</b> качества...</p>` |

### 4.6. Дубликат SKU — перезапись

При создании нового товара система автоматически проверяет, существует ли товар с таким же SKU.

Если дубликат найден:
1. Появится окно **«Товар уже существует»** с информацией о существующем товаре (название, handle, SKU)
2. Доступны два действия:
   - **«Отмена»** — вернуться к редактированию формы
   - **«Перезаписать»** — удалить старый товар и создать новый с текущими данными

## 5. Таб «Импорт» — массовый импорт из JSON

Позволяет загрузить несколько товаров одновременно из JSON-файла.

### 5.1. Подготовка JSON-файла

Нажмите **«Скачать шаблон JSON ↓»** для получения файла-примера со всеми полями.

Файл должен содержать массив товаров. Допустимы два формата:

**Формат 1 — объект с ключом `products`:**
```json
{
  "products": [
    {
      "title": "Кулирка пенье белая",
      "sku": "00101",
      "price": 850,
      "fabric_type": "kulirka",
      "composition": "100% хлопок",
      "width_cm": 180,
      "measurement_unit": "running_meter",
      "inventory": 30000,
      "color": "Белый",
      "color_hex": "#FFFFFF",
      "quality": "Пенье",
      "country": "Турция"
    }
  ]
}
```

**Формат 2 — массив на верхнем уровне:**
```json
[
  {
    "title": "Кулирка пенье белая",
    "sku": "00101",
    "price": 850,
    "fabric_type": "kulirka",
    "composition": "100% хлопок",
    "width_cm": 180
  }
]
```

### 5.2. Загрузка и валидация

1. Нажмите на область загрузки и выберите `.json` файл
2. Система автоматически проверит:
   - Корректность JSON-формата
   - Наличие всех обязательных полей у каждого товара
   - Корректность значения `fabric_type`
   - Положительность цены
   - Корректность `measurement_unit` (если указан)
   - Отсутствие дублирующихся SKU в файле
3. При ошибках — отобразится жёлтый блок **«Найдены проблемы»** со списком

### 5.3. Предпросмотр и импорт

Если валидация пройдена:
1. Отобразится таблица **«Предпросмотр»** со всеми товарами из файла
2. Проверьте данные визуально
3. Нажмите зелёную кнопку **«Импортировать N товаров»**
4. Дождитесь завершения (спиннер + прогресс)

### 5.4. Результаты импорта

После завершения отображается:
- **Счётчики:** «N Создано» (зелёный) и «N Ошибки» (красный)
- **Детальный список** по каждому товару:
  - ✅ — успешно создан (с сообщением «Создан успешно»)
  - ❌ — ошибка (с описанием причины)

> **Дубликаты:** Товары с уже существующим SKU или handle будут **пропущены** (не перезаписаны). Для обновления существующих товаров используйте редактирование через вкладку «Товары».

После импорта доступны кнопки:
- **«Загрузить новый файл»** — сбросить и загрузить другой файл
- **«Открыть каталог →»** — посмотреть результат на сайте

## 8. Правила расчёта цен и запасов

### Цена

| Единица | Что вводите | Как хранится в Medusa |
|---------|-------------|----------------------|
| **Погонный метр** | Цена за 1 метр в рублях (напр. `1100`) | Хранится как есть (1100). Фронтенд делает обратную конвертацию |
| **Штука** | Цена за 1 штуку в рублях (напр. `500`) | Умножается на 100 → хранится `50000` (в копейках) |

### Запас

| Единица | Что вводите в форме | Что указывать в JSON | Как хранится |
|---------|--------------------|--------------------|-------------|
| **Погонный метр** | Значение в **сантиметрах** (30000 = 300 м) | В **сантиметрах** | Как есть |
| **Штука** | Количество штук | Количество штук | Как есть |

> **Примеры для метровых тканей:**
> - 300 метров → вводите `30000`
> - 50 метров → вводите `5000`
> - 1.5 метра → вводите `150`

### Отображение в таблице товаров

- Цена метровых: отображается `N ₽/м`
- Цена штучных: отображается `N ₽/шт` (делится на 100 для перевода из копеек)
- Запас метровых: отображается `N м` (делится на 100 для перевода из см)
- Запас штучных: отображается `N шт`

## 9. Частые вопросы и ошибки

### Общие

| Проблема | Решение |
|----------|---------|
| Страница `/admin` не загружается | Убедитесь, что фронтенд запущен (`npm run dev` в папке `frontend`) |
| «Неверный email или пароль» | Проверьте учётные данные. Бэкенд Medusa должен быть запущен на порту 9000 |
| Список товаров пуст после создания | Нажмите **«↻ Обновить»** — список загружается один раз при открытии вкладки |

### Создание / редактирование товара

| Проблема | Решение |
|----------|---------|
| «Укажите название / SKU / цену...» | Заполните все обязательные поля, отмеченные звёздочкой (*) |
| Фото не загружается: «превышает 300 КБ» | Сожмите изображение на сервисе [iloveimg.com](https://www.iloveimg.com/ru/compress-image) |
| Товар не появился в каталоге | Проверьте, что товар имеет статус «Опубликован». Обновите страницу каталога |
| При редактировании нельзя изменить SKU | Это ожидаемое поведение — SKU является уникальным идентификатором и не подлежит изменению |

### Импорт из JSON

| Проблема | Решение |
|----------|---------|
| «JSON должен содержать массив товаров...» | Проверьте формат файла: массив объектов `[{...}]` или `{ "products": [{...}] }` |
| «Отсутствует поле...» | Добавьте все обязательные поля: `title`, `sku`, `price`, `fabric_type`, `composition`, `width_cm` |
| «Недопустимый fabric_type» | Используйте только значения из [справочника](#6-справочник-типы-тканей) |
| «Товар с SKU уже существует. Пропущен.» | При импорте дубликаты пропускаются. Для обновления используйте редактирование |
| «Дублирующийся SKU в файле» | Два товара в одном файле имеют одинаковый SKU — исправьте |

### Советы

- **Шаблон**: Скачайте шаблон JSON с вкладки «Импорт» — он содержит пример со всеми полями
- **Handle**: Если не указать `handle`, он будет создан автоматически из транслитерации названия
- **Скидки**: Бейдж скидки отображается на карточке товара в каталоге, только если указан `discount_percent`
- **Обложка**: Первое загруженное изображение автоматически становится обложкой товара
- **Массовое создание**: Для добавления более 5 товаров используйте импорт из JSON — это быстрее
